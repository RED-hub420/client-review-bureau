File a review

Verified businesses can submit a documented review tied to a transaction, attempted transaction, delivery, chargeback, nonpayment, or another marketplace issue.

For businesses

Businesses can request a public listing, apply for bureau membership, and maintain response rights on published files connected to their marketplace activity.

How CRB handles reviews

Structured like a bureau, not a social feed.

Reviews move through intake, screening, notice, response, and publication or closure. The goal is a usable marketplace record with status history attached to the same file.

1

Review is filed

A verified business submits a review tied to a real marketplace relationship and provides the basic file details.

2

Bureau screens the matter

CRB checks the filing for fit, abusive language, duplicate activity, unsupported accusations, and publication readiness.

3

Notice is sent

When a matter is accepted for handling, the reported party can be notified and given a response window.

4

Status is updated

Files may show as pending review, answered, unresolved, unanswered, dispute open, or closed.

5

Public file remains attached

Later updates, correction requests, and closure notes remain on the same file so readers can follow the record trail.

What appears in a public file

Directory use
  • Published summary of the review matter
  • Current bureau status and last activity date
  • Response history when one has been received
  • 3-year file totals and basic issue categories

What does not appear

Protected
  • Payment account details, identity numbers, or raw processor notices
  • Unscreened uploads, private notes, or internal handling comments
  • Medical details, unrelated personal information, or doxxing-level data
  • Any statement presented as a court finding or legal determination
