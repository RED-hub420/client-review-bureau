Member reporting standards

These rules keep the network usable. Records must stay factual, narrow, and tied to a real transaction.

Current ruleset Applies to all members

1. Who may open records

Only verified member businesses may submit records. Each account must be tied to a real operating business, named contact, and approved use category. Anonymous submissions are not accepted.

2. What a record must include

Records must identify the event date, work order or invoice reference, file type, and a clear description of what occurred. When available, members should include supporting documents, signed approvals, delivery records, route logs, processor notices, or service records.

3. What is not allowed

  • Insults, threats, or retaliatory language
  • Unsupported accusations of criminal conduct
  • Irrelevant personal information or doxxing
  • Protected medical, financial, or identity details not needed for the file
  • Speculation about motive, character, or lifestyle

4. Visibility rules

Records may be saved as internal-only, pending review, or restricted member visible depending on support level, moderation outcome, and dispute status. Visibility can be reduced during review.

5. Corrections and disputes

If a challenged file enters dispute review, visibility may be reduced until the member provides additional support or the file is corrected, withdrawn, or locked. Status changes stay attached to the original file trail.

6. Why the rules are tight

Member files can affect how another business handles deposits, releases, scheduling, pickup, delivery, or payment terms. That is exactly why the system has to stay narrow and evidence-based.

Review checklist

Internal
  • Confirm the business is active and verified.
  • Check file type fit and visibility level.
  • Review attached support and redactions.
  • Reduce visibility if dispute threshold is met.
  • Lock file after final determination.

Rule objective

Reference

The point is not public shaming. The point is a narrow, auditable record system for documented transaction loss, dispute history, and repeat issue tracking inside a verified business network.
